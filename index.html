welcome to META VERSE ROBOTICS - First Decentralized Meta Verse Robotic Project

Our main prupose is to use the whole creativity of th world. Every token holder will be able to vote for the future of the project. We can see voting for new features on the UGV's or UAV's as an example. There will be an pool of ideas wich can be integrated to the meta verse robotics, the idea wich will get the most votes will be developed by our team, in the future even the developers of the MVR will be voted by the folk.

Quality

Transparency

Every code on the project will be written in ROS (Robotic Operation System), and every one will have access to the codes after the intermediate relases over github. To run the codes Ubuntu 20.04 operating system will be required wich can be installed on every personal pc.

Data Collection

The data to use when building the enviroment and specific characteristics of the robots will be gathered from the internet the availabile free data sets from universitys. Since ROS wirtten codes will work on real machines real test will be possible.

Designing

For the desing of the simulation eviroment open source models of Gazebo are used at the first stage, in future there can be an seperate team only for desinging the enviroment. Our aim is to create the future of meta verse robotic together.

Development

The development will be made by our Developer team, befor the releases no codes will be published public. After the releases every one can see the released sections of the code on git but the codes will be licenced by MVR and only for non commercial use available.

Quality

The written codes and the simulation enviroment are tested precisely by our team. We guarantee that the simulation enviroment will work on every pc if the necessary hardware and software is provided. The aim is to bring this simulation enviroment to an common platform.

Release

The releases will made on the dates that are specified after the voting process, of course ther can be delays due to the development in this cases all token holders will be payed back in the money amount the lose after the deadline but these are the further steps.

About MVR Token

MVR project is the clef to connect realworld robotics applications with meta verse.

All codes that can operate in the simulation enviroment are appropriate to work on real machines, the jackal machine used in the fist enviroment can seen in the prev the yellow UGV, on the side we can the real machine all the codes can be uploaded to these real machines and tested in real world if desired.

If there are people who want to test the codes in real world, the can load and run the codes on these vechiles they will only need to change the namspaces and load the correct drivers. in the picture there is an 2-d lazer added to the Jackal UGV wich is not included in the simulation enviroment created by us, but it can be in the next steps.

  • Community will work on meta verse, users can test in real.

Only token holders will able to vote !

The decision how the future of the robotics in meta verse will be created is going to made by the token holders. After the initial promotions the contract for voting progress will be launched and the folk will vote for the options how the simulation enviromnet should be in the next steps.

  • The worth of the vote will be propotional to the token amount.
  • Only accounts that had an amount of tokens BEFORE the selection can vote.
  • Vote only once.
  • An seperate web site is in building progress for the selection.

Build the future of robotics together, stay tuned.

How voting will work? #1

Every month there will be an voting progress, after the first voting has started. Evrey token holder will be able to vote even if they have one MVR. But before voting beggins the token balance of the accounts will be recorded tokens transffered after the voting starts will not alter the selection. The voting will ahve an duration of 1 day. The token holder folk will vote for the topic that can be integrated to the enviroment. If we give an example: "Map the enviroment with an UAV (Drone) and deliver an cargo to a location" this could be an possible selection topic for wich token holder can vote.

Who can vote? #2

All token balances before the selction beginn will be recorded, after that every related token holder cann connect their wallet to the voting web site and vote. Be careful only the balance befor the selection begins will be taked into account. İf the token balance until the second selection does not change it will not alter the situation every holder can vote as normal. Every vote is worth the token balance before the selection begins.

Benfit of the Project? #3

The most important benefit of the project is that every people on the world will make an decision on the future of robotics in Meta Verse no class distinction between people, but the main point is that in future only people that are holding the token will be able to vote.
